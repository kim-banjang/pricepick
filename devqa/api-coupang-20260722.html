한국 운영 담당자가 공유한 테스트 API입니다. 정체는 한국 담당자 측 제휴 대행사(ddaynet)의 링크 생성 API입니다 — 대행사가 보유한 쿠팡 파트너스 계정으로 경유 링크·광고 소재를 생성하며, click_id·adid 파라미터로 클릭을 추적·연결합니다. 정식 쿠팡 파트너스 API는 앱 패키지네임이 확정된 후 신청·발급받을 예정입니다. 그 전까지는 이 테스트 API로 개발을 진행합니다.Đây là API test do người phụ trách vận hành Hàn Quốc chia sẻ. Bản chất là API tạo link của công ty đại lý liên kết (ddaynet) phía Hàn Quốc — dùng tài khoản Coupang Partners của đại lý để tạo link đi qua·creative quảng cáo, và theo dõi·liên kết click qua tham số click_id·adid. API Coupang Partners chính thức sẽ được đăng ký·cấp phát sau khi package name của app được chốt. Trước đó, phát triển dựa trên API test này.
[참조][Tham khảo]테스트 API 호출 시 응답 = 쿠팡 파트너스 광고 소재 HTML(ads-partners.coupang.com/openapi/image 등 이미지 리소스 포함). click_type=2 파라미터로 클릭 추적 방식을 지정. 주요 파라미터: code(대행사 사이트코드) · click_type · adid · click_id.Response khi gọi API test = HTML creative quảng cáo Coupang Partners (bao gồm resource ảnh dạng ads-partners.coupang.com/openapi/image v.v.). 가능한 범위: '쿠팡 바로가기' 경유 이동 + 클릭 추적(click_id) — 실제 쿠팡으로 이동하는 것까지 확인됐습니다. 불가능한 범위: 구매 확정 통보(포스트백) 수신입니다. 포스트백을 받으려면 우리 쪽에 수신 서버가 필요하고, 이건 정식 연동 단계의 몫입니다. CMS의 포스트백 로그·수익 분석 화면을 '미확인' 처리해 둔 기존 결정과 일치하는 내용입니다.Phạm vi làm được: Chuyển hướng đi qua 'Coupang bắt đầu' + theo dõi click (click_id) — đã xác nhận chuyển đến Coupang thực tế. Phạm vi chưa làm được: Nhận thông báo xác nhận mua hàng (postback). Để nhận postback cần server nhận riêng phía chúng ta, đây là phần thuộc giai đoạn tích hợp chính thức. Khớp với quyết định trước đó đã đánh dấu 'chưa xác nhận' cho màn Log postback·Phân tích doanh thu ở CMS.
