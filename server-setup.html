쿠팡 연동 시 주의사항Nếu tích hợp Coupang

쿠팡 API를 호출하는 순간 응답 지연은 서버가 아닌 쿠팡 쪽에서 옴.
외부 API 연동 작업(쿠팡 픽 구매 처리, 기프티콘 발급, 이메일/SMS 발송 등)은 반드시 백그라운드 워커(Worker Queue)로 분리해 처리할 것.Đây mới là thứ đáng quan tâm. Hệ thống sẽ gọi: App → Backend → Coupang API → Backend → Database. Độ trễ chủ yếu đến từ Coupang. Không phải server.
RabbitMQ 또는 Redis Streams 사용 권장 — 유저가 결과를 기다리지 않게 비동기 처리Do đó nên: RabbitMQ / Redis / Background Worker — để tránh người dùng phải chờ.

처음부터 잡아야 할 것 (나중에 갈아엎기 방지)Những việc nên làm ngay từ đầu

  • 🏗
    Stateless API 설계 — 나중에 백엔드 서버를 여러 대로 늘릴 수 있게 서버가 상태를 저장하지 않는 구조로 설계Thiết kế theo hướng Stateless API, để dễ dàng nhân bản nhiều backend sau này.
  • ⚡
    Redis 적극 활용 — 캐시, 세션, 레이트 리밋, 티켓·포인트 정보 등 자주 읽는 데이터 캐싱Dùng Redis cho cache, session (nếu cần), rate limiting và các dữ liệu truy cập nhiều như thông tin ticket/point.
  • 🔄
    외부 연동은 백그라운드 워커로 — 쿠팡, 기프티콘, 이메일/SMS 등 외부 API 호출은 RabbitMQ 또는 Redis Streams로 큐잉Tách các tác vụ đồng bộ với hệ thống bên ngoài (Coupang, Gifticon, gửi email/SMS) sang Background Worker thông qua hàng đợi (RabbitMQ hoặc Redis Streams).
  • 📊
    로깅 · 모니터링 표준화 — 처음부터 Prometheus + Grafana(또는 동급) 도입Chuẩn hóa logging và monitoring (Prometheus + Grafana hoặc tương đương) ngay từ giai đoạn đầu.
  • 💾
    DB 자동 백업 — 매일 자동 백업 + 주기적 복구 테스트Thiết lập backup tự động cho cơ sở dữ liệu hằng ngày và kiểm tra khả năng khôi phục định kỳ.
  • 🚀
    CI/CD 구축 — 빠른 배포 + 업데이트 리스크 최소화Chuẩn bị sẵn quy trình CI/CD để triển khai nhanh và giảm rủi ro khi cập nhật.

Đây là lộ trình hợp lý giúp tối ưu chi phí đầu tư trong giai đoạn đầu nhưng vẫn sẵn sàng đáp ứng tăng trưởng trong tương lai.
