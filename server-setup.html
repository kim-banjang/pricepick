쿠팡 연동 시 주의사항

쿠팡 API를 호출하는 순간 응답 지연은 서버가 아닌 쿠팡 쪽에서 옴.
외부 API 연동 작업(쿠팡 픽 구매 처리, 기프티콘 발급, 이메일/SMS 발송 등)은 반드시 백그라운드 워커(Worker Queue)로 분리해 처리할 것.
RabbitMQ 또는 Redis Streams 사용 권장 — 유저가 결과를 기다리지 않게 비동기 처리

처음부터 잡아야 할 것 (나중에 갈아엎기 방지)

  • 🏗
    Stateless API 설계 — 나중에 백엔드 서버를 여러 대로 늘릴 수 있게 서버가 상태를 저장하지 않는 구조로 설계
  • ⚡
    Redis 적극 활용 — 캐시, 세션, 레이트 리밋, 티켓·포인트 정보 등 자주 읽는 데이터 캐싱
  • 🔄
    외부 연동은 백그라운드 워커로 — 쿠팡, 기프티콘, 이메일/SMS 등 외부 API 호출은 RabbitMQ 또는 Redis Streams로 큐잉
  • 📊
    로깅 · 모니터링 표준화 — 처음부터 Prometheus + Grafana(또는 동급) 도입
  • 💾
    DB 자동 백업 — 매일 자동 백업 + 주기적 복구 테스트
  • 🚀
    CI/CD 구축 — 빠른 배포 + 업데이트 리스크 최소화
