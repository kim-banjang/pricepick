1. 한 줄 정의1. Định nghĩa một dòng

포스트백 로그 = 제휴몰(쿠팡·11번가·G마켓·알리익스프레스·아이허브)이 "우리 회원이 방금 그쪽에서 결제를 끝냈다"고 우리 서버에 알려준 통보(콜백) 기록. 회원이 PricePick 화면에서 제휴몰로 넘어가 물건을 사면, 그 제휴몰이 우리 서버로 "누가 얼마에 샀다"는 신호를 보내주는데 그게 포스트백이다. 이 신호가 들어와야 비로소 가지급(적립예정) 티켓 처리와 제휴 수수료(매출) 집계가 시작된다 — 이 화면은 그 신호들이 실제로 들어오고 있는지, 얼마가 잡히고 있는지를 확인하는 로그다.Log Postback = bản ghi thông báo (callback) mà cửa hàng liên kết (Coupang·11번가·G마켓·AliExpress·iHerb) gửi cho server của mình để báo "hội viên của mình vừa thanh toán xong bên đó". Khi hội viên từ màn PricePick chuyển sang cửa hàng liên kết và mua hàng, cửa hàng đó sẽ gửi tín hiệu về server mình báo "ai đã mua với giá bao nhiêu" — đó chính là postback. Phải có tín hiệu này thì xử lý vé tạm cấp (chờ tích lũy) và tổng hợp hoa hồng liên kết (doanh thu) mới bắt đầu được — màn này là log để xác nhận tín hiệu đó có thực sự vào không, và số tiền ghi nhận là bao nhiêu.

4. 왜 지금은 검증이 안 되는가4. Vì sao hiện chưa xác nhận được

쿠팡을 예로 들면, 지금 개발에서 쓰는 API는 한국 운영 담당자가 공유한 제휴 대행사(ddaynet)의 링크 생성 API다. 이 API로 '쿠팡 바로가기' 이동과 클릭 추적(click_id)까지는 확인됐지만, 구매 확정 통보(포스트백) 수신은 아직 안 된다 — 포스트백을 받으려면 우리 쪽에 수신 서버가 필요하고, 이건 정식 쿠팡 파트너스 API 연동 단계의 몫이다. 정식 API는 앱 패키지네임이 확정된 후 신청·발급받을 예정이다. (devqa 쿠팡 API 답변, 2026-07-22)Lấy Coupang làm ví dụ, API đang dùng để phát triển hiện tại là API tạo link của công ty đại lý liên kết (ddaynet) do người phụ trách vận hành Hàn Quốc chia sẻ. Với API này đã xác nhận được việc chuyển hướng 'Coupang bắt đầu' và theo dõi click (click_id), nhưng chưa nhận được thông báo xác nhận mua hàng (postback) — để nhận postback cần có server nhận riêng phía mình, đây là phần thuộc giai đoạn kết nối API Coupang Partners chính thức. API chính thức sẽ đăng ký·cấp phát sau khi package name của app được chốt. (Trả lời devqa API Coupang, 2026-07-22)

함께 '미확인'으로 전환된 화면Các màn cũng chuyển sang 'Chưa xác nhận'
같은 이유(포스트백 실데이터 없음)로 수익 분석·매출 내역·정산 내역 세 화면도 함께 '미확인' 상태다. 세 화면 다 포스트백 데이터가 있어야 숫자가 의미를 갖기 때문 — 화면 자체(데모 연출 데이터)는 그대로 두고 상태만 바꾼 것이다.Vì cùng lý do (chưa có dữ liệu postback thật), 3 màn Phân tích doanh thu·Lịch sử doanh thu·Lịch sử quyết toán cũng đang ở trạng thái 'Chưa xác nhận'. Vì cả 3 màn đều cần dữ liệu postback thì số liệu mới có ý nghĩa — chỉ đổi trạng thái, còn màn hình (dữ liệu demo dàn dựng) vẫn giữ nguyên.
